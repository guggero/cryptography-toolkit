BIP-352: Silent Payments

Explanation

BIP-352 Silent Payments let a receiver publish one static address (sp1q...) while every actual payment lands on a unique, unlinkable taproot output. The price: the receiver has to scan the chain, computing one ECDH multiplication per eligible transaction.

This page performs that scan entirely in your browser. A block-dn server provides per-transaction tweak data (input_hash·A, one 33-byte point per eligible transaction) plus a taproot-only compact block filter. For every transaction the page derives the candidate output keys for your address (and its change label), matches them against the filter, and only downloads full blocks on a hit — where it then identifies your outputs across all output indexes k per BIP-352.

Privacy: the scan private key and spend public key never leave this page; the server only observes which block ranges are downloaded. Note: only the base address and the change label (m=0) are scanned; other labels are not supported yet. On mainnet, tweak data is 40–100 MB per 2000 blocks — a full scan since taproot activation downloads several GB.

Sources:

  • BIP-352 Specification
  • btcutil-js Library (scan engine)
  • block-dn (tweak data + custom filters over HTTP)
