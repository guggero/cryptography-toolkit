BIP-322: Generic Signed Message Format

Explanation

BIP-322 defines a generic signed message format for Bitcoin that supports all script types (P2WPKH, P2TR, P2SH-P2WPKH, P2PKH, multisig, etc.).

Unlike the legacy signmessage RPC which only works with P2PKH addresses, BIP-322 works by constructing a virtual "to-spend" transaction and signing it, proving ownership of the address.

The simple format (for native segwit: P2WPKH, P2TR) encodes just the witness stack. The full format (for P2SH-P2WPKH, P2PKH) encodes the complete signed virtual transaction.
